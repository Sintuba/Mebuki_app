## 基本方針

- **シンプルさ優先**: 過度な抽象化・早期最適化を避ける
- **型安全**: `any` は使わない。`unknown` 経由でキャストする
- **コンポーネント分割**: 1コンポーネント = 1責務

## TypeScript

```ts
// NG: any
const data: any = fetchData()

// OK: unknown
const data: unknown = fetchData()
if (typeof data === 'string') { ... }
```

## React

- Server Component をデフォルトとする
- `'use client'` は必要な最小範囲だけ
- `useEffect` の依存配列は必ず正確に書く

## ファイル命名

| 種別 | 規約 |
|---|---|
| コンポーネント | `PascalCase.tsx` |
| ユーティリティ | `camelCase.ts` |
| ページ | `page.tsx`（App Router 規約） |

## Git

- `feat:` / `fix:` / `update:` / `chore:` プレフィックスを使う
- 1コミット = 1変更単位

## スタイリング

- Tailwind クラスは `className` に直接記述
- CSS ファイルは `globals.css` のみ
- カスタムカラーは CSS 変数（`--color-*`）で定義
