## Tailwind CSS v4 変更点と注目ポイント

Tailwind CSS v4への移行を検討する上で、把握しておくべき主要な変更点と、特に注目すべき新機能・改善点をまとめたメモです。

### v3 → v4 の主な変更点

Tailwind CSS v4では、設定の簡素化とCSSとのネイティブな統合が強化されています。

-   `tailwind.config.js` ファイルが基本的に不要になった（CSS ファイル内で直接設定）
-   プロジェクトのCSSファイル内で `@import "tailwindcss";` を使ってTailwindを読み込む形式に統一
-   カスタムテーマや設定は、CSS変数（カスタムプロパティ）を使って直接CSS内で定義可能に

### CSS ベースの設定例

設定がCSSファイル内で完結するようになり、より直感的になりました。

```css
@import "tailwindcss";

/* カスタムテーマの定義例 */
@theme {
  --color-brand: #22c55e; /* ブランドカラーの定義 */
  --font-size-xs: 0.75rem; /* フォントサイズの定義 */
  /* その他のカスタムプロパティもここに定義 */
}

/* その他のグローバルなスタイル定義など */
```

### 注目すべき新機能・改善点

v4では、開発体験の向上やより柔軟なカスタマイズを可能にする新機能が導入されています。

-   **CSS変数参照クラスのサポート**: `border-border` のように、直接CSS変数を参照するクラス名が利用可能になり、より動的なスタイル定義が可能に。
-   **自動ダークモード対応**: `bg-background` や `text-foreground` といったセマンティックなクラス名を通じて、OSのテーマ設定に応じたダークモードが自動的に適用されるようになり、実装の手間が軽減されます。
-   **Vite プラグインとの統合改善**: Vite開発環境でのパフォーマンスと開発体験がさらに向上しました。

### 今後の調査項目

上記変更点や新機能について、さらに以下の点を詳しく調査し、ベストプラクティスを確立していく必要があります。

-   CSS変数ベースのテーマ定義の詳細と複雑な設定への応用
-   既存プロジェクトからのv4への移行パスと注意点
-   `tailwind.config.js` を利用する場合のユースケース（より高度なカスタマイズなど）
-   各新機能の具体的なパフォーマンスへの影響と利用例
